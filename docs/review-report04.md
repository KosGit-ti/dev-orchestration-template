### 3-2. SCAN_DIRS 行番号の評価（連鎖的誤りの訂正）

**Explore エージェントの出力（review-report02・03 が依拠した情報）**：

```
22→REPO_ROOT = Path(__file__).resolve().parent.parent
24→# スキャン対象ディレクトリ…
25→SCAN_DIRS = [
29→]
```

**Read ツールによる実測値**：

```
20→REPO_ROOT = Path(__file__).resolve().parent.parent
22→# スキャン対象ディレクトリ…
23→SCAN_DIRS = [
27→]
```

Explore エージェントの出力は**実際のファイルと2行ずれていた**。

この誤りにより、review-report02.md と review-report03.md は以下の誤った評価を行った：

| レポート | 誤った評価 | 実際 |
|---|---|---|
| review-report02.md | 「L22 はコメント行、L23-27 が本体、微差」 | 正しい（ただし L22 はコメント行であり評価自体は偶然正しい） |
| review-report02.md | 「定義本体は L23-27」 | 実際は L23-27 ✅（これは正しかった） |
| review-report03.md | 「L22 = REPO_ROOT 代入文（無関係）」 | 実際 L20 = REPO_ROOT、L22 = コメント行 ❌ |
| review-report03.md | 「開始・終端ともに3行ずれ」 | 実際は `:22-26` vs L22-27 で終端のみ1行ずれ ❌ |

**結論**：

`review-report01.md` の `SCAN_DIRS: ci/policy_check.py:22-26` という記述は：
- L22（コメント行）〜L26（最終要素）を指す表記として実質的に正確
- 欠落しているのは L27 の閉じ括弧 `]` のみ（終端が1行ずれ）

review-report02.md が「コメント行を含む微差」と評価したのは**結果的には正しかった**が、
review-report03.md が「L22 = REPO_ROOT の無関係代入文、3行ずれ」と訂正したのは**誤り**だった。
Explore エージェントの誤出力に基づいて正確な評価を誤った評価で上書きしてしまった。

## 5. 改善策

### 5-1. review-report03.md の訂正が必要な箇所

| 箇所 | 誤った記述 | 正しい記述 |

### 5-2. 検証プロセスの改善

今回の連鎖的誤りは、Explore エージェント（サブプロセス）の出力と実際のファイルに
2行のズレが生じたことを Read ツールで確認せずに踏襲したことが原因である。

| 場面 | 現行の問題 | 改善策 |
|---|---|---|
| 行番号の引用 | Explore エージェントの出力をそのまま使用 | Read ツールで直接確認してから引用する |
| スニペットの正確性 | 「誰かが確認済み」という累積前提に依拠 | 各レポートで独立して Read ツールを使用する |
| 評価の訂正 | 新しいスニペットで旧評価を上書きするが、新スニペット自体を検証しない | 訂正根拠となるスニペットも同時に独立検証する |
