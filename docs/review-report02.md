### 3-2. bootstrap.sh のファイル存在条件（説明不足）

`review-report01.md §3` では「bootstrap.sh 実行後に ci.yml を丸ごと上書き生成する」と記述されている。
この事実自体は正確だが、生成の前提条件についての説明が不足している。

`bootstrap.sh` L258 の実際の条件：

```bash
if [ -f "$CI_FILE" ] && [ "$LANGUAGE" = "python" ]; then
```

- **`[ -f "$CI_FILE" ]` はファイル存在チェック**（ファイルが存在するときのみ生成する）。
- テンプレートをクローンした直後は `ci.yml` が存在するため、通常のフロー（clone → bootstrap）では正しく動作する。
- `ci.yml` が存在しない状態で `bootstrap.sh` を実行した場合、CI ファイルは**生成されない**。

つまり：

| 実行時の ci.yml 状態 | bootstrap.sh の動作 |
|---|---|
| 存在する（通常のクローン直後） | Python 用 ci.yml に上書き生成される ✅ |
| 存在しない（削除後などの異常状態） | 生成されず CI が機能しない ❌ |

`review-report01.md` の記述は通常フローにおいて正確だが、ファイル存在を前提とする
この条件の意図と動作については説明が欠落している。
